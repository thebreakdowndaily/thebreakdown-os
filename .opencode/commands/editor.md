# THE BREAKDOWN
## Editor Command v1.0 — Quality Control

### Mission
Review the story against editorial standards before publication.

### Editorial Checklist

#### Structure
- [ ] All 7 required sections present and in order
- [ ] Opening answers: what happened, why now, why it matters
- [ ] Evidence section has h3 sub-sections for each discrete claim
- [ ] Data section has at least 3 data points (table, chart, or stats card)
- [ ] Debate section has at least 2 competing positions
- [ ] What's Next has 3 scenarios with probabilities totalling 100%
- [ ] Primary Sources has ≥10 entries with URLs

#### Fact-Check
- [ ] Every claim tagged with evidence type
- [ ] Every data point has a source URL
- [ ] Every confidence score is between 0-100
- [ ] No unlabeled opinions presented as fact
- [ ] All quotes attributed to named sources
- [ ] Comparative claims have baselines

#### Writing
- [ ] No passive voice in lead paragraph
- [ ] No "could", "might", "possibly" without uncertainty label
- [ ] Every paragraph conveys one distinct fact
- [ ] Reading time estimate matches content length

#### Technical
- [ ] Slug is kebab-case, unique
- [ ] Tags are lowercase, max 12
- [ ] Hero image has caption with credit
- [ ] JSON-LD metadata complete
- [ ] Story link works in /explained/ and /story/ pages

### Scoring
- Pass: All critical items checked
- Conditional: Minor fixes needed (list them)
- Fail: Missing evidence or structural issues
