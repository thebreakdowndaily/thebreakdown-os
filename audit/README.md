# Audit Framework

## Project Status: Version 1.0.0 (Frozen)

The Audit-as-Code platform has reached Version 1.0.0 and the core contracts are now frozen. It has transitioned from a set of isolated checks to a mature, coherent auditing platform with a deterministic execution model, governance, and domain-specific capabilities.

| Phase | Description |
|------|--------------|
| **Phase A** | Core framework, aggregator, and directory layout |
| **Phase B/C** | Implement framework SDK, policy loader, test runners |
| **Phase D** | Dependency graph (Kahn's algorithm), Architecture Plugin, Operations Plugin, Security Plugin |
| **Phase E** | Accessibility, Performance, SEO, Knowledge Graph, Editorial Plugins |
| **Phase F** | Telemetry separation, Platform Health Score, Registry generation, V1 Freeze |

## Purpose
The **Audit-as-Code** system provides a deterministic, **read-only** verification pipeline for The Breakdown platform. It ensures that engineering, security, performance, accessibility, SEO, knowledge-graph, and operational health meet the standards defined in the platform's governance documents.

## Compatibility Statement

The Version 1.0 platform explicitly supports the following boundaries:

| Component | Required Version | Support Policy |
|-----------|-----------------|----------------|
| **Node.js** | `>= 24.x` | Supported on LTS and Active releases. |
| **Plugin SDK** | `1.0.0` | Frozen for v1.x. Additive changes only in minor releases. Breaking changes require v2.0.0. |
| **Manifest Schema** | `2.0` | Frozen for v1.x. |
| **Report Schema** | `1.0` | Output schema is guaranteed stable for downstream CI/CD consumption. |
| **Policy Schema** | `1.0` | Frozen. Policy rules can evolve, but the structure of policy definitions is stable. |

*Any changes to these core components require a Major version bump.*

## Core Capabilities (Plugins)

The platform evaluates the repository using the following repository-centric plugins:

1. **Architecture** (`architecture-audit`) - Validates repository layout and manifest structure.
2. **Operations** (`operations-audit`) - Verifies CI/CD, lockfiles, and package manager policies.
3. **Security** (`security-audit`) - Scans for exposed secrets and structural security headers.
4. **Accessibility** (`accessibility-audit`) - Ensures automated a11y testing tools and configuration are present.
5. **Performance** (`performance-audit`) - Validates lighthouse and bundle-analyzer setups.
6. **SEO** (`seo-audit`) - Checks for `robots.txt`, `sitemap.xml`, and core metadata tags.
7. **Knowledge Graph** (`knowledge-graph-audit`) - Validates the canonical registry and schema structures.
8. **Editorial** (`editorial-audit`) - Evaluates Knowledge Objects for Editorial Constitution compliance.

## Platform Health Score (PHS)

The aggregator computes an overarching **Platform Health Score (PHS)** representing the unweighted average score across all non-mock plugins. This provides a clear, trackable metric for overall repository health. 
In Version 1, both `platformHealthScore` and `weightedPlatformHealthScore` are provided in the report to ensure backward compatibility for future configurable weightings.

## Running Audits

```bash
# Execute the full audit suite locally
npx tsx audit/scripts/audit.ts

# Generate the authoritative plugin registry
npx tsx audit/scripts/generate-registry.ts
```

The audit script automatically resolves the plugin dependency graph, executes plugins deterministically, and writes the results to `audit-report.json`.
