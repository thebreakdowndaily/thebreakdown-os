## 1. Identifier Grammar

- Human‑readable constituency research IDs follow the pattern `UP-AC-001`.
- Derived IDs may be expressed as `UP-AC-001-CLAIM-000001` for claims, `UP-AC-001‑SOURCE‑00001` for sources, etc.
- These identifiers are **not** database primary keys; they serve as stable, human‑readable references.

## 3. Lifecycle & State Transitions

### Candidate Lifecycle
```
CANDIDATE → PROPOSED → APPROVED → REJECTED (terminal)
```
### Issue Lifecycle
```
PENDING → ASSIGNED → IN_REVIEW → RESOLVED | DISMISSED (terminal)
```
### Promise Lifecycle
```
PROMISED → FULFILLED → BROKEN (terminal)
```
### Project Lifecycle
```
IDEATION → PLANNED → EXECUTING → COMPLETED → ARCHIVED (terminal)
```
### Verification Lifecycle
```
PENDING → EDITOR_VERIFIED → SOURCE_VERIFIED → AI_REVIEWED → REJECTED | ARCHIVED
```
### Contradiction Lifecycle
```
UNRESOLVED → RESOLVED (terminal)
```
### Financial Stage Flow
```
ANNOUNCEMENT → BUDGET_PROVISION → ADMIN_APPROVAL → FINANCIAL_SANCTION →
FUNDS_RELEASED → TENDER_VALUE → CONTRACT_AWARD → PAYMENT →
REPORTED_EXPENDITURE → UTILIZATION_REPORTED → PHYSICAL_PROGRESS →
PHYSICALLY_COMPLETE → OPERATIONAL → FINAL_COST → OUTCOME
```
*Stages may be omitted if not applicable, but **no stage may be inferred** from a later stage.*

## 4. Evidence Model

- **Source Reliability** – captured via `SourceTier`.
- **Evidence Relationship** – captured via `EvidenceRelationship` (`SUPPORTS`, `CONTRADICTS`, `QUALIFIES`, `SUPERSEDES`, `CONTEXTUALIZES`).
- **Claim Confidence** – captured via `ConfidenceLevel` (A‑E).

## 5. Provenance

Every source must record:
- Canonical ID (if available)
- URL (persistent, archived if possible)
- Issuing authority/publisher
- Title
- Publication/issue date
- Access date
- Page/section/table reference
- Preserved/archive reference (e.g., Wayback Machine)
- Relevant excerpt or structured extraction
- Extraction method (manual, AI, OCR, etc.)
- Researcher/agent responsible for extraction
- Verification state (`VerificationStatus`)

## 6. Negative Evidence

- `NOT FOUND` **≠** `DID NOT HAPPEN`.
- Use `UNKNOWN` or `UNVERIFIED` status when records are missing.

## 7. Geographic Attribution

Attributes may apply to:
- Single constituency, multiple constituencies, village, ward, block, municipality, district, parliamentary constituency, region, statewide.
- For multi‑constituency projects, record the **total** amount once and list affected geographies; **do not duplicate** the amount per constituency.

## 8. Financial Methodology

Track each stage explicitly; do **not** infer missing stages. Record:
- Original vs revised cost
- Original vs revised deadline
- Cost overrun, time overrun
- Financial year
- Funding authority/source
- Implementing authority
- Multi‑year and multi‑constituency considerations

## 9. Responsibility Model

Separate attribution of:
- Legal authority
- Administrative authority
- Funding authority
- Implementing authority
- Regulatory authority
- Political advocacy/influence
- Public credit claim

Political credit (e.g., MLA inauguration) does **not** imply funding or implementation responsibility.

## 10. Entity Resolution

Rules for reconciling duplicate or variant entities:
- Exact name match → same entity.
- Transliteration variants (Hindi ↔ English) → alias list.
- Renamed constituencies → maintain historical alias.
- Political‑party abbreviations → map to canonical name.
- Department reorganisations → retain lineage via `aliases` field.
- Contractors with similar names → require manual verification; do **not** merge by fuzzy similarity alone.
