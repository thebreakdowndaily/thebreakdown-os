# Product Decision Records (PDRs)

Product decisions for The Breakdown Knowledge Platform are recorded here.

Every significant UX or product decision gets a PDR. This creates traceability from reader problem to shipped feature.

## Template

```markdown
# PDR-XXX — Title

## Problem
What reader problem does this solve?

## Decision
What are we building?

## Alternatives Considered
What else did we think about?

## Chosen Because
Why this approach over the alternatives?

## Governing Documents
Which specs, contracts, or standards authorize this?

## Success Metrics
How will we know it worked?

## Deliberately Out of Scope
What are we choosing not to do?
```

---

## Rules

1. A PDR is created before any implementation code is written.
2. Every PDR must answer all four qualifying questions before entering design.
3. PDRs are lightweight — a good PDR is 15–30 lines.
4. PDRs are not architecture documents. They record product reasoning.
