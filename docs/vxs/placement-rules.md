# Placement Rules

> **VXS v1.0 (Frozen)** — Where visuals appear in relation to narrative flow. These rules govern the spatial and sequential relationship between text and visual elements.

---

## 1. Visual-Narrative Relationship Types

Every visual has exactly one relationship to the surrounding text:

| Type | Description | When to use |
|------|-------------|-------------|
| **Anchoring** | Visual precedes text and establishes context | The visual is the best entry point to the topic |
| **Embedded** | Visual appears within the relevant text block | The visual illustrates a specific claim being made |
| **Sequential** | Multiple visuals in a fixed order | A concept requires multiple visual steps |
| **Reference** | Visual is linked from text but positioned in a sidebar or panel | The visual is supplementary — useful but not essential to the narrative |
| **Summary** | Visual follows text and consolidates | The visual reinforces or reviews what the text explained |

## 2. General Placement Rules

### Rule 1 — One visual per 400 words minimum

A chapter without visuals is a wall of text. A chapter with too many visuals is a gallery, not a narrative. The minimum density is one visual per 400 words of narrative. If a chapter has no visuals in a 400-word stretch, either a visual is missing or the section needs structural review.

Exception: Sections with extended primary-source quotations or historiographical debate may go 600 words without a visual when the text itself is the evidence.

### Rule 2 — Visuals are part of the narrative, not interruptions

A visual must never interrupt a paragraph mid-argument. Place visuals at natural paragraph breaks, section transitions, or before/after a claim is introduced, not in the middle of developing a point.

### Rule 3 — The first visual is always Anchoring

A chapter's opening visual establishes the topic, tone, and stakes. It must be an Anchoring visual that appears before the first heading after the chapter title. Choose the category based on the chapter's central question:

| If the chapter primarily answers... | Opening visual category |
|------------------------------------|------------------------|
| "Where did this happen?" | Orientation |
| "How did this happen?" | Explanation |
| "What was it like?" | Context |
| "When did this happen?" | Timeline |
| "Proof?" | Evidence |

### Rule 4 — Each major claim gets one Embedded visual

For every registered claim of High or Medium confidence, at least one Embedded visual should support it. The visual cannot replace the evidentiary requirement (the claim must still cite its sources) but should provide a visual dimension to the evidence.

### Rule 5 — Orientation precedes the narrative it supports

When a chapter discusses geographic space (a border, a migration, a battle), the Orientation visual must appear before the narrative that refers to locations on that map. The reader must be oriented before the orientation is useful.

### Rule 6 — Comparison accompanies the specific claim it supports

A Comparison visual showing quantities must appear alongside the paragraph that discusses that data. It must not appear in a separate "Data" section or appendix. Evidence proximity applies to visuals as much as to text.

### Rule 7 — Sequential visuals share a single block

When two or more visuals form a sequence (e.g., before/after maps, a process diagram with steps), they must be placed in a single visual block with a shared caption. The reader experiences them as one narrative unit, not separate images.

### Rule 8 — No visual islands

Every visual must be preceded by at least one sentence that prepares the reader for it ("The following map shows...") and followed by at least one sentence that interprets it ("What this shows is that..."). A visual dropped into text without introduction or interpretation is an island.

## 3. Placement by Visual Category

### Orientation

| Placement | Notes |
|-----------|-------|
| Anchoring — before geographic narrative begins | The first time a location is mentioned, the map must have already appeared |
| Embedded — alongside the claim the data supports | Thematic maps visualise claims |
| Sequential — in a map sequence (reference → thematic → flow) | Orientation first, then thematic layers, then flows |

Do not place Orientation visuals in:
- Sidebars on first appearance (must be in narrative flow)
- After the geographic narrative they support has already begun

### Evidence

| Placement | Notes |
|-----------|-------|
| Embedded — alongside the claim the document supports | The facsimile must appear when the document is first discussed |
| Reference — sidebar when full text is needed | If the document is long, show the facsimile in a referenced panel with a key excerpt in the narrative |

Do not place Evidence visuals in:
- The chapter opening (use Context or Orientation instead)
- The learning section (documents are evidence, not study aids)

### Explanation

| Placement | Notes |
|-----------|-------|
| Anchoring — before a section describing a multi-step process | The reader sees the whole process before reading about its parts |
| Embedded — alongside the final step when the process is cumulative | Use when the process builds toward a conclusion |
| Summary — learning section when used as study aid | Study aids are review tools placed in the learning section |

Do not place Explanation visuals in the middle of a step. The diagram must show all steps or the full sequence.

### Context

| Placement | Notes |
|-----------|-------|
| Anchoring — chapter open | The most historically significant photograph opens the chapter |
| Embedded — within a relevant claim | The photograph must directly illustrate the claim |
| Summary — learning section | A key photograph may be repeated in the learning summary |

Do not place Context visuals in:
- Sidebars or panels (they belong in the narrative flow)
- Groups of more than 3 without compelling sequential logic
- The middle of a paragraph

### Comparison

| Placement | Notes |
|-----------|-------|
| Embedded — alongside the claim it supports | Charts make numerical claims visible; they must appear with the claim |
| Summary — learning section | Key comparison charts may appear in the learning summary |

Do not place Comparison visuals in:
- A separate "Data" section or appendix
- Before the claim they support is introduced

### Timeline

| Placement | Notes |
|-----------|-------|
| Anchoring — after the opening scene, before the narrative begins | The timeline orients the reader in time before the chapter proceeds |
| Reference — sidebar or collapsible panel | A timeline that covers the full chapter scope may be referenced rather than embedded |

Do not place a Timeline after a section that has already described the events.

### Network

| Placement | Notes |
|-----------|-------|
| Anchoring — before a section with multiple entities | Use when introducing 4+ named entities that interact |
| Reference — sidebar when the diagram is complex | If the diagram has 10+ nodes, position it in a referenced panel |

Do not place Network visuals after the entities have already been introduced.

## 4. Visual-Caption Relationship

### Rule 1 — Captions are analytical, not descriptive

A caption must explain what the visual means, not what it is. "A map of Punjab, 1947" is descriptive. "The Radcliffe Line divided Punjab through its most densely populated and religiously mixed districts, separating Lahore from Amritsar and creating one of the largest refugee flows in history" is analytical.

### Rule 2 — Captions do not duplicate text

The caption should not repeat a sentence from the surrounding narrative. If the best caption for a visual is the sentence next to it, the visual and text are redundant and one should be removed.

### Rule 3 — Caption length by category

| Category | Caption length | Notes |
|----------|---------------|-------|
| Orientation | 1–3 sentences | What the map shows + how to read it |
| Evidence | 1–3 sentences | What the document is + why it matters |
| Explanation | 2–4 sentences | Process or concept + key decision points |
| Context | 2–4 sentences | Historical significance + what the photograph depicts |
| Comparison | 2–4 sentences | Pattern + data source + significance |
| Timeline | 1–3 sentences | Period + key events |
| Network | 1–3 sentences | Structure + key relationships |

### Rule 4 — Captions include metadata

Every caption must include at minimum:
- Source attribution (archive, collection, or data source)
- Date (of creation for photographs/documents; of data for maps/charts)
- Evidence level for archival material

## 5. Visual Density Limits

| Content type | Maximum visuals | Notes |
|-------------|----------------|-------|
| Chapter (800–1,500 words) | 4–6 | 1 anchoring, 3–5 embedded |
| Chapter (1,500–3,000 words) | 6–10 | 1 anchoring, 5–9 embedded |
| Chapter (3,000–5,000 words) | 10–15 | 1 anchoring, 1 timeline, 8–13 embedded |
| Chapter (5,000+ words) | 15–20 | + sequential blocks as needed |
| Investigation | 20–30 | + supplementary panels |
| Learning section | 2–4 | Study aids (Explanation), summary charts (Comparison), one Context anchoring |

### Category distribution guideline

A chapter should not rely on a single category. Distribute across at least three categories:

| Chapter type | Minimum categories | Example mix |
|-------------|-------------------|-------------|
| Geographic focus (border, migration) | 4 | Orientation + Context + Comparison + Timeline |
| Policy or institutional | 3 | Explanation + Network + Timeline |
| Biographical | 3 | Context + Evidence + Timeline |
| Event-driven (crisis, war) | 4 | Context + Orientation + Timeline + Comparison |
