# Post-Publication Review

**Version:** 1.0
**Status:** Operational

Every published chapter undergoes a structured review after publication. The review captures what worked, what did not, and what the institution should do differently for the next chapter.

---

## Timing

- **Initial review:** 7 days after publication
- **Full review:** 30 days after publication
- **Follow-up:** 90 days after publication (if significant changes occurred)

## Review components

### 1. Reader feedback

- [ ] What reader feedback has been received? (corrections, questions, praise, criticism)
- [ ] Are there patterns in the feedback?
- [ ] Did readers engage with the evidence panels?
- [ ] Did readers use the glossary?
- [ ] Did readers follow continue-learning links?

### 2. Correction submissions

- [ ] How many correction submissions were received?
- [ ] How many were accepted?
- [ ] What categories of corrections were they? (factual, interpretive, typographical, source-related)
- [ ] Did any correction require a change to the canonical knowledge layer?

### 3. Trust metrics

- [ ] What is the chapter's Trust Score?
- [ ] How does it compare to the institutional average?
- [ ] Has the Trust Score changed since publication? (if so, why?)

### 4. Pipeline performance

- [ ] How long did the chapter spend in each pipeline stage?
- [ ] Which stages had gate failures?
- [ ] How many revision cycles were needed?
- [ ] Was the estimate accurate?
- [ ] What caused delays?

### 5. Evidence quality

- [ ] Was the evidence sufficient to support the claims?
- [ ] Were there evidence gaps that became apparent after publication?
- [ ] Did any source prove unreliable after publication? (source decay)

### 6. Narrative quality

- [ ] Did the chosen story pattern work?
- [ ] Were the Four Layers distinguishable?
- [ ] Was the prohibited language audit clean?
- [ ] Did the chapter answer all Six Questions?

### 7. Learning effectiveness

- [ ] Were the learning objectives achieved?
- [ ] Did readers engage with the reflection prompts?
- [ ] Was the glossary used?
- [ ] Did continue-learning recommendations lead to other chapters?

## Review format

The Post-Publication Review is a written document, no more than two pages, containing:

```
Chapter: <title>
Published: <date>
Review date: <date>
Reviewer: <name>

1. What worked
   - (list 3–5 things that went well)

2. What did not work
   - (list 3–5 things that went poorly or could improve)

3. Process changes needed
   - (specific changes to the editorial workflow, style guide, or review process)

4. Lessons for the next chapter
   - (what the team should do differently)

5. Follow-up actions
   - (items with owners and dates)
```

## Action items

Every review produces action items. Each action item has:

- Owner (the person responsible)
- Description (what needs to be done)
- Deadline (when it needs to be done)
- Status (open, in progress, completed)
- Verification (how we know it is done)

## Institutional memory

The Post-Publication Review is stored in the Book of Record. It is referenced in the next chapter's planning to avoid repeating mistakes.

## Quarterly synthesis

Every quarter, the Editor-in-Chief synthesises all Post-Publication Reviews from that quarter into a Quarterly Editorial Review. Patterns across chapters are identified. Systemic process changes are proposed. The synthesis is reviewed at the Monthly Editorial Review.
