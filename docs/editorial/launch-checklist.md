# Launch Checklist

**Version:** 1.0
**Status:** Operational

The Founding Edition launch is a single event. It publishes the institutional package — the method, the standards, and the first chapter — in a specific sequence that builds trust before presenting conclusions.

---

## Sequence

The launch follows this order, not the order of completion. Each item prepares the reader for the next.

### Phase 1 — Public Methodology (T-4 weeks)

- [ ] `/methodology` page is drafted and reviewed
- [ ] `/methodology` explains: how research is conducted, how claims are verified, how confidence scores work, why scholarly disagreements are surfaced, how corrections are handled, how AI is used and not used, how chapters evolve through versions, how visual assets are selected and verified
- [ ] `/methodology` is reviewed by Editor-in-Chief
- [ ] `/methodology` is published at canonical URL
- [ ] `/methodology` is linked from site footer

### Phase 2 — Editorial Constitution (T-3 weeks)

- [ ] Editorial Constitution is formatted for public reading
- [ ] Version number and governance hierarchy are displayed
- [ ] Link to `/methodology` is included
- [ ] Editorial Constitution is published at canonical URL
- [ ] Editorial Constitution is linked from `/methodology` and site footer

### Phase 3 — Trust Dashboard (T-2 weeks)

- [ ] `/trust` page is built and populated
- [ ] Trust Dashboard displays: Editorial Constitution version, number of published chapters, total claims registered, primary sources cited, scholarly disagreements documented, corrections issued, average Trust Score, evidence debt, Gold Standard Review completion status, last verification date
- [ ] Institutional Trust Index is computed and displayed
- [ ] `/trust` is reviewed by Editor-in-Chief
- [ ] `/trust` is published at canonical URL
- [ ] `/trust` is linked from `/methodology`, site footer, and every chapter

### Phase 4 — Supporting Documents (T-1 week)

- [ ] Corrections Policy is published
- [ ] Sources Policy is published
- [ ] Transparency Statement is published
- [ ] All supporting documents link to `/methodology` and `/trust`
- [ ] All supporting documents are linked from site footer

### Phase 5 — Chapter 1 (Launch day)

- [ ] Chapter 1 passes full Publication Checklist
- [ ] Human approval is documented
- [ ] Chapter 1 is published at canonical URL
- [ ] Chapter 1 is listed in Volume I index
- [ ] Chapter 1 is discoverable through search
- [ ] Chapter 1 is linked from `/trust`, `/methodology`
- [ ] `/trust` is updated with Chapter 1 data
- [ ] Knowledge Graph is updated with Chapter 1 node

### Phase 6 — Founding Edition Announcement (Launch day + 1)

- [ ] Founding Edition announcement is prepared
- [ ] Announcement includes: what was published, what the Editorial OS is, the institutional commitment to transparency, the invitation for informed critique
- [ ] Links to `/methodology`, Editorial Constitution, `/trust`, and Chapter 1
- [ ] Announcement is published on the platform
- [ ] Announcement is distributed through available channels

---

## Pre-Launch Verification

### Evidence readiness

- [ ] Chapter 1 has passed Gold Standard Review (all 7 phases)
- [ ] External peer review is complete — outcomes documented
- [ ] All changes resulting from external review are applied
- [ ] Changes not made (and why) are documented and published alongside the chapter

### Institutional readiness

- [ ] Corrections workflow is tested (submit a test correction, verify it routes correctly)
- [ ] Living update workflow is tested
- [ ] Human approval gate is tested (verify the release does not proceed without sign-off)
- [ ] Editor-in-Chief has confirmed readiness in writing

### Technical readiness

- [ ] All URLs resolve
- [ ] All pages render in all three reading modes
- [ ] All links between methodology, constitution, trust dashboard, and chapter are functional
- [ ] Search returns all published pages for relevant queries
- [ ] Load times are acceptable
- [ ] Mobile rendering is functional

### Rollback plan

- [ ] A rollback procedure is documented: unpublish chapter, revert Knowledge Graph, post status notice
- [ ] The Editor-in-Chief has the authority to initiate rollback without additional approval
- [ ] Maximum acceptable time to rollback is 30 minutes

---

## Post-Launch Observation

After launch, the Editorial Office enters an observation period (minimum 2 weeks, no maximum). During this period:

- No new chapters are published
- The office monitors: reader feedback, correction submissions, trust dashboard metrics, load and performance data
- The office documents: what readers ask about, which claims generate questions, which sections are revisited, which explanations are confusing
- The office compiles a post-launch review with recommendations for the editorial process

The observation period ends when the Editor-in-Chief determines that the institution has learned enough from the launch to improve the process for Chapter 2.
