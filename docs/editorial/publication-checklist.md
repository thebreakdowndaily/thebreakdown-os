# Publication Checklist

**Version:** 1.0
**Status:** Operational

Run this checklist before every chapter publication. The Editorial Conductor verifies each item. The Editor-in-Chief signs off. The human approval gate does not close until every item passes.

---

## Content

- [ ] Every factual claim is traced to an evidence item in the Claim Registry
- [ ] Confidence scores are assigned and visible
- [ ] All Six Questions are answered in the narrative
- [ ] The Four Layers (What Happened / What the Evidence Shows / Where Historians Disagree / Why It Matters) are distinguishable throughout
- [ ] Competing interpretations are attributed to named scholars or schools
- [ ] Counterarguments appear at the point of disagreement, not collected at the end
- [ ] Prohibited language audit is complete — zero occurrences
- [ ] Executive summary is 3–5 paragraphs answering the Six Questions

## Evidence

- [ ] Claim density meets standard: 50+ claims per chapter
- [ ] Evidence items meet standard: 120+ per chapter
- [ ] Sources cited meet standard: 100+ per chapter
- [ ] Primary source ratio: at least 40% of citations are to primary sources
- [ ] Every claim in the Claim Registry has at least one evidence link
- [ ] Sources are verified and confidence tiers are assigned
- [ ] Evidence gaps are documented (published as known limitations)

## Visual Assets

- [ ] Hero image selected with documented provenance
- [ ] Every map, chart, and document has complete provenance (archive, shelfmark, license, credit)
- [ ] License status confirmed for every visual (public domain, CC, fair use, licensed)
- [ ] AI-generated images are labelled as such
- [ ] Maps follow the Land and Maritime Boundaries Policy (disputed boundaries use dashed lines)
- [ ] Alt text is written for every image
- [ ] Captions are fact-checked and sourced
- [ ] Pedagogical purpose is stated for every visual (what does this image teach?)
- [ ] Resolution is adequate for publication

## Metadata and SEO

- [ ] Title is 60 characters or fewer
- [ ] Description is 155 characters or fewer
- [ ] Canonical URL is set and resolves
- [ ] Open Graph title, description, image, and type are populated
- [ ] Twitter Card is set (summary_large_image)
- [ ] Structured data (Article schema) validates against schema.org
- [ ] Publication date is set
- [ ] Last verified date is set
- [ ] Version label is set (semantic versioning)
- [ ] Reading time is calculated by mode
- [ ] Difficulty rating is assigned
- [ ] Primary topic and subtopics are tagged

## Knowledge Graph

- [ ] Chapter node is created in the Knowledge Graph
- [ ] Edges exist to: collection, volume, entities, thinkers, concepts, claims
- [ ] Bidirectional links are verified (entity pages reference this chapter)
- [ ] Related chapters include "see also" references
- [ ] Cross-references to this chapter from previously published chapters are updated

## Internal Links

- [ ] Every entity mention links to the correct entity page
- [ ] Every thinker mention links to the correct thinker profile
- [ ] Every claim reference links to the correct registry entry
- [ ] Every source citation links to the correct source entry
- [ ] Every "continue learning" recommendation links to an existing knowledge object
- [ ] Zero broken internal links

## Learning Section

- [ ] Learning objectives are specific and verifiable
- [ ] Key questions are restated with answers from the narrative
- [ ] Glossary terms are defined for a non-specialist reader
- [ ] Further reading is curated (5–7 entries, each with a one-sentence explanation)
- [ ] Reflection prompts test understanding, not recall
- [ ] Continue-learning recommendations link to existing published objects

## Accessibility

- [ ] Images have alt text
- [ ] Colour is not the only means of conveying information
- [ ] Heading hierarchy is logical (h1 → h2 → h3, no skips)
- [ ] Link text is descriptive (not "click here")
- [ ] Tables have headers
- [ ] Reading mode toggle functions correctly

## Technical

- [ ] Chapter renders in all three reading modes (explorer, scholar, researcher)
- [ ] Evidence panels render correctly
- [ ] Table of contents navigates correctly
- [ ] All images load
- [ ] Mobile rendering is functional
- [ ] Load time is within acceptable range (no single asset > 500KB unoptimised)
- [ ] No JavaScript errors in console

## Final Sign-Off

- [ ] Version number is assigned (follows semantic versioning)
- [ ] Change log entry is written (for first publication: "Initial publication")
- [ ] Trust Score is updated
- [ ] Corrections policy link is included
- [ ] Methodology link is included
- [ ] Human approval is documented (named editor, timestamp, explicit authorisation)
