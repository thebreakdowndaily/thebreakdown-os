# Monthly Editorial Review

**Version:** 1.0
**Status:** Operational

The Monthly Editorial Review is the institution's quality check. Every month, the Editorial Office pauses to assess whether the operating system is producing trustworthy, high-quality knowledge objects at a sustainable pace.

---

## Timing

The Monthly Editorial Review occurs on the last working day of each month. Duration: 2 hours.

---

## Agenda

### 1. Institutional Trust Index (30 minutes)

Review every component of the Institutional Trust Index:

| Component | Current score | Previous score | Change | Notes |
|---|---|---|---|---|
| Evidence Quality (25%) | | | | |
| Primary Source Ratio (15%) | | | | |
| Review Completion (15%) | | | | |
| Freshness (10%) | | | | |
| Correction Quality (10%) | | | | |
| Transparency (10%) | | | | |
| Reader Trust (10%) | | | | |
| Expert Review (5%) | | | | |
| **Total (100%)** | | | | |

**Action:** If the total has dropped below 80/100, publishing stops until the index recovers. The Editor-in-Chief announces the recovery plan.

### 2. Pipeline performance (30 minutes)

- **Throughput:** How many chapters moved through each stage this month?
- **Cycle time:** Average time per stage. Any stage getting slower?
- **Gate pass rate:** What proportion of stages passed on first attempt?
- **Bottlenecks:** Which stage is the slowest? Is it a capacity problem or a process problem?
- **Failure patterns:** Are there recurring gate failures? Which gates fail most often?

### 3. Correction analysis (20 minutes)

- **Volume:** How many corrections were received this month?
- **Accept rate:** How many were accepted?
- **Categories:** Factual, interpretive, typographical, source-related — what is the distribution?
- **Cascade rate:** How many corrections affected multiple chapters?
- **SLA compliance:** Were correction SLAs met?
- **Patterns:** Is there a systemic source of errors?

### 4. Evidence debt (15 minutes)

- **Current evidence debt:** Total claims with insufficient evidence
- **Changes:** Has debt increased or decreased?
- **Oldest debt:** Claims that have been in debt the longest
- **Plan:** What is being done to reduce debt?

### 5. Process improvements (15 minutes)

- **Improvements implemented this month:** What changed?
- **Improvements proposed:** What should change next month?
- **Measurement:** How will we know if the improvement worked?

### 6. Forward look (10 minutes)

- **Next month's priorities:** What must be accomplished?
- **Risks:** What might go wrong?
- **Capacity:** Do we have the people we need?

---

## Outputs

The Monthly Editorial Review produces:

1. **Trust Index report** — current scores, changes, and trends
2. **Pipeline performance report** — cycle times, pass rates, bottlenecks
3. **Correction analysis** — volume, patterns, systemic issues
4. **Evidence debt report** — current state and reduction progress
5. **Process improvement plan** — what will change before next month
6. **Decision record** — all decisions made during the review, recorded in the Book of Record

## Publication

The Trust Index report is published on the `/trust` dashboard. The Pipeline Performance report is internal. The Correction Analysis is published on the `/trust` dashboard (aggregate, not individual correction details).

## Escalation

If the Institutional Trust Index drops below 80/100 during the month (not just at the review), the Editorial Conductor escalates immediately. The Monthly Review becomes a recovery planning session rather than a routine review.
